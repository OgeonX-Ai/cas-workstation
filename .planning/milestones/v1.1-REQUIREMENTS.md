# Requirements: CAS Loop Engineering v1.1

**Defined:** 2026-07-04
**Core Value:** A Windows-first developer can trust CAS to pursue repository goals in parallel without losing control of state, cost, safety, evidence, or completion.

## v1.1 Requirements

### CI and Security

- [x] **CI-01**: Active portfolio workflows use minimum required `GITHUB_TOKEN` permissions.
- [x] **CI-02**: Active jobs have proportionate `timeout-minutes` limits and correct CodeQL language configuration.
- [x] **CI-03**: Third-party GitHub Actions are pinned or covered by an explicit automated update policy.

### Contract Registry

- [x] **REG-01**: CAS Contracts builds and validates a deterministic registry containing stable and immutable schema paths.
- [x] **REG-02**: Genuine contract consumers verify the expected registry schema or manifest in CI without requiring secrets.
- [x] **REG-03**: Registry availability, digest, and compatibility failures produce actionable CI evidence.

### Robustness

- [x] **ROB-01**: Bicep repositories enforce secure network defaults and repository lint configuration.
- [x] **ROB-02**: Backlog findings C3-C6 are reproduced before modification and receive regression coverage when confirmed.
- [x] **ROB-03**: False-positive robustness findings are explicitly dismissed with concrete code or test evidence.

### Verification

- [x] **VER-01**: Every changed repository passes its native build, test, lint, and security checks.
- [x] **VER-02**: Contract publication and consumer validation work end to end from generated registry artifact to consumer gate.
- [x] **VER-03**: v1.1 milestone audit and UAT report no unresolved blocking requirement or integration flow.

## Future Requirements
