  <behavior>
    - Given a branch whose content is fully present in origin/default (squash-merged): `git diff origin/<default> <branch>` is EMPTY -> gate returns SAFE-TO-DELETE.
    - Given a branch with real unmerged content: tree diff is NON-EMPTY -> gate returns RETAIN with the --stat as evidence (fail-closed: never delete on non-empty).
    - Given a branch and a THREE-DOT diff that is non-empty but the two-dot tree diff is empty: gate uses the two-dot result (SAFE-TO-DELETE) — this is the squash-awareness contract.
    - Default branch is resolved per-repo, not hardcoded (Promptimprover=master, others=main).
  </behavior>
  <action>
    Write scripts/squash-aware-branch-gate.ps1 (ASCII-only — PS 5.1 ANSI hazard; no em-dashes or curly quotes). Params: `-RepoPath`, `-Branch`, `-DeleteSafe` (switch, default off = report-only), `-Json`. Logic: fetch/prune, resolve default via `git -C <repo> symbolic-ref refs/remotes/origin/HEAD --short` (strip `origin/`), then run `git -C <repo> diff --stat origin/<default> <branch>` (TWO-dot direct tree compare — explicitly NOT `origin/<default>...<branch>`). Empty output => disposition SAFE-TO-DELETE; non-empty => RETAIN with the stat text. Only when `-DeleteSafe` AND SAFE-TO-DELETE, run `git -C <repo> branch -D <branch>` (branch is squash-merged so -d would refuse; -D is correct HERE because the tree gate already proved content parity). Emit a pscustomobject per branch (Repo, Branch, Default, Disposition, Evidence) and `-Json` mode for machine consumption. Add a comment block explaining WHY two-dot not three-dot.
    Write Pester tests in scripts/tests/squash-aware-branch-gate.Tests.ps1: build throwaway local repos in a temp dir, simulate a squash-merge (default branch gets the content as one commit; feature branch keeps the multi-commit history), assert the gate returns SAFE-TO-DELETE; build a genuinely-diverged branch and assert RETAIN. Cover the three-dot-vs-two-dot divergence case explicitly.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local branch history -> delete decision | Squash-merge makes history lie about mergedness; deleting on the wrong signal loses work. |
| worktree -> removal | A worktree may hold unpushed commits that exist nowhere else. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-07 | Tampering | deleting a branch whose content is not actually in default | mitigate | Fail-closed tree gate: delete ONLY on empty two-dot `git diff origin/<default> <branch>`; non-empty => RETAIN. Proven by Pester. |
| T-38-08 | Repudiation | branch history diverged (squash) misread as unmerged work | mitigate | Two-dot direct tree compare instead of three-dot merge-base diff; documented in the script and report. |
| T-38-09 | Denial of Service (data loss) | removing a worktree with unpushed commits | mitigate | `git rev-list --count @{u}..HEAD` guard before removal; > 0 => preserve + report, never remove. |
| T-38-10 | Tampering | touching foreign dirty files during cleanup | accept | Task removes only the two named worktrees and the 6 named branches; foreign dirty files untouched (Agent-Hierarchy rule 3). |
</threat_model>
