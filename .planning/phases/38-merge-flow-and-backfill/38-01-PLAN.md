<objective>
Stand up the merge-flow mechanism that lets low-risk PRs (dependabot version bumps and docs-only changes) merge to their default branch within hours on green checks, WITHOUT weakening two-party review and WITHOUT agent self-approval.

The chosen mechanism (design rationale in docs/merge-flow-policy.md, this plan): a dedicated **review-bot GitHub App** with its own credentials that approves a PR only when (a) a fail-closed diff classifier confirms the PR is in the auto-merge-eligible class, (b) the deterministic autogen critic_cli reports 0 blocking findings, and (c) CI is green. GitHub auto-merge then lands it. Everything outside the class falls back to operator review.

Purpose: close REQ-1.5.1 — flow for the safe class, human depth for the rest, a real second party throughout.
Output: two org workflows, the classifier script, the branch-protection-as-code script, the policy doc, and a proven auto-merge of a test in-class PR.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authoring agent -> PR | Untrusted content proposing a merge; the agent must never be able to approve or merge it. |
| PR diff -> auto-merge decision | The classifier + critic run on attacker-influenceable content (the diff itself). |
| review-bot credentials -> merge authority | The App can approve+merge; compromise = unreviewed code lands. |
| third-party actions / autogen checkout -> CI runtime | Supply-chain surface for the approval workflow. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | Elevation | auto-merge merging code no human reviewed | mitigate | Class boundary is the entire safety argument: fail-closed classifier (Task 1) derives IN-CLASS from actual changed paths, denylist forces any executable/workflow/dependency file OUT-OF-CLASS -> human path. Documented honestly in merge-flow-policy.md. |
| T-38-02 | Spoofing | PR mislabels itself as docs-only to gain auto-merge | mitigate | Classifier ignores labels/title entirely; decision from `gh pr view --json files` changed paths only. |
| T-38-03 | Spoofing | authoring agent impersonates the review bot to self-approve | mitigate | Approval requires the App installation token (secret-gated, not available to agent sessions); permission layer already denies agent self-approval (Agent-Hierarchy rule 2). |
| T-38-04 | Tampering | review-bot approves a diff that fails deterministic policy | mitigate | Approve step is unreachable unless classifier IN-CLASS AND critic_cli exit 0 AND CI green — three fail-closed gates in sequence. |
| T-38-05 | Repudiation | who/what merged is unclear | mitigate | App identity is a distinct login in the PR timeline; workflow logs the classifier + critic output. |
| T-38-06 | Elevation | review-bot App over-scoped | mitigate | App granted only PRs R/W, Contents R/W, Checks R, Metadata R; no admin, no secrets scope. |
| T-38-SC | Tampering | third-party action / autogen critic supply chain in the approval path | mitigate | actions/create-github-app-token and the autogen checkout pinned by commit SHA; critic_cli sourced from a pinned autogen ref, not `latest`. |
</threat_model>
