## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| root repo direct-push -> master | An unreviewed path into the workspace root; agents or scripts could land unreviewed changes. |
| solo admin + enforce_admins | Protection that locks out the only admin bricks the flow; too-loose protection is theater. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-38-11 | Elevation | root repo direct-push bypasses two-party review | mitigate | Branch protection requires 1 approving review on master; direct pushes to master blocked. |
| T-38-12 | Spoofing | authoring agent self-approves a root PR | mitigate | CODEOWNERS reviewer is the operator (non-agent) identity; permission layer denies agent self-approval (Agent-Hierarchy rule 2). |
| T-38-13 | Denial of Service | enforce_admins locks the solo admin out of emergency fixes | mitigate | Documented enforce_admins temp-relax/restore break-glass in the runbook (relax + restore in the same sitting, verify `.enabled`). |
| T-38-14 | Repudiation | undocumented root posture drifts back to direct-push | mitigate | Applied as-code via apply-branch-protection.ps1 and recorded in merge-flow-policy.md; re-runnable and auditable. |
</threat_model>
