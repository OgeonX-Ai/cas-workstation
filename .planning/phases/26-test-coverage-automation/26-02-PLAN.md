<objective>
Wire the existing autogen (Python 3.12 / pytest) test suite into an enforced BRANCH
coverage CI gate, fix the undeclared coverage dependency, and close the measured gap
toward 100% branch on the active MAF orchestration/agent-boundary code.

Purpose: autogen's CI already runs `pytest --cov` but `pytest-cov` is NOT declared in
requirements.txt (fragile — relies on a transitive/preinstalled package), coverage is
LINE-only (no `branch=true`), and the enforcement step reads `line-rate` against an
unreachable 100% so the gate is either red or accidentally passing. This plan declares
pytest-cov, adds a `.coveragerc` enabling branch coverage with boilerplate/legacy
exclusions, switches the gate to a ratcheted `--cov-fail-under` branch threshold, and
generates meaningful tests to raise the measured branch coverage.

Output: updated requirements.txt, new .coveragerc, rewritten coverage step in ci.yml,
and new/expanded tests under tests/.
</objective>
