enforced BRANCH-coverage CI gate, and close the measured gap toward 100% branch on

meaningful tests to raise the branch-rate.

and new/expanded test files that raise measured branch coverage.

|----------|-------------|
| PR author -> CI gate | Untrusted test code crosses into the coverage measurement; author can attempt to satisfy the number without testing behavior |
| coverlet output -> gate logic | XML coverage report is parsed and drives the merge signal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Tampering | New xUnit tests | mitigate | Acceptance criteria require every new test to contain meaningful assertions on observable behavior (state, telemetry, exception type); reviewer rejects assertion-free coverage padding |
| T-26-02 | Tampering | coverlet.runsettings exclusions | mitigate | ExcludeByAttribute/ExcludeByFile limited to generated/boilerplate globs and standard attributes; excluding business logic to inflate the number is out of bounds and reviewable in the diff |
| T-26-03 | Denial of Service | Flaky tests in CI | mitigate | New tests must be deterministic (NSubstitute mocks, no live API/network, no real clock/sleep); baseline rounded DOWN to 4 decimals to avoid float-comparison flake |
| T-26-SC | Tampering | NuGet installs | accept | No new NuGet packages introduced (coverlet.collector already pinned at 10.0.1); no package-legitimacy checkpoint required |
</threat_model>
