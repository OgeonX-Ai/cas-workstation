export const canonicalSchemaBase = "https://schemas.coding-autopilot.dev";

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| schema $id -> consumer fetch | Consumers (cas-evals, future autopilot-core/gsd-orchestrator integrations) treat $id as both identity and fetch location; a mismatch or unannounced change silently breaks offline validators and live fetchers alike |
| CI runner -> GitHub Pages | `pages.yml` build job fetches nothing external but publishes to a public, unauthenticated origin; anyone can read the registry (intended — it's a public schema registry) |
| git tag -> release content | `build-registry.mjs` trusts `git archive` output for whatever tag exists; a maliciously pushed tag could inject unexpected schema content into an "immutable" release path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Repudiation | $id rewrite (this plan) | mitigate | CHANGELOG.md BREAKING entry + PR body explicitly document the reversal of v1.1.1's decision and cite the live `gh api .../pages` evidence, so the rationale is traceable and not silently re-reverted a third time |
| T-32-02 | Tampering | schema files ($id field) | accept | Schema files are plaintext JSON reviewed via PR diff; no automated tooling writes $id except this manual edit — human PR review is the control |
| T-32-03 | Information Disclosure | public GitHub Pages registry | accept | Registry is intentionally public (schemas contain no secrets); this is the product, not a leak |
| T-32-04 | Denial of Service | consumers hardcoding old $id (cas-evals contracts.py) | mitigate | Addressed directly in plan 32-02: cas-evals' hardcoded `$id` string checks are updated in the same milestone track, and the CHANGELOG breaking-change note gives any other future consumer explicit warning |
| T-32-SC | Tampering | npm ci in verify commands | accept | No new package.json dependencies added by this plan; existing lockfile/CI already governs supply chain for cas-contracts' ajv/ajv-formats deps |
