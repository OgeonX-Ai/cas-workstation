    - Test 3 (convergence field present): the `-Json` output object always includes a
      `convergence` field, set to `"BASELINE"` on the very first run (Test 1) and to either
      `"CONVERGING"` or `"NOT_YET_CONVERGING"` on any subsequent run (Test 2), computed by
      comparing this cycle's `newFindings.Count` against this cycle's `closedItems.Count`
      (`CONVERGING` when new < closed).

  Parsing: read the backlog file's lines; track the most recent `## ` or `### ` heading seen as
  each row's `Section`; for each line matching a table row whose first cell is a 1-3 letter +
  digits ID (see `<interfaces>`), record `{Id, Section, RawRow}` where `RawRow` is the full
  trimmed row text (used as a content fingerprint, not just presence/absence). Skip non-matching
  rows silently (headers, prose, the ID-less "-" row) - a parse miss must never throw.

  Snapshot/diff: load the most recent snapshot JSON already present in `-SnapshotDir` (by
  filename date, if any exist) as the "previous" item set, keyed by `Id`. Compute `newFindings`
  (Ids present now, absent from previous), `closedItems` (Ids present in previous, absent now),
  and `unchangedCount`. If no previous snapshot exists, every current Id is a `newFinding`,
  `closedItems` is empty, and `convergence` is `"BASELINE"`; otherwise `convergence` is
  `"CONVERGING"` when `newFindings.Count < closedItems.Count`, else `"NOT_YET_CONVERGING"`.

  Write the dated snapshot: `{SnapshotDir}\backlog-survey-{yyyy-MM-dd}.json` containing
  `captured_at` (ISO 8601), `backlogPath`, and the full current `items` array
  (`Id`, `Section`, `RawRow`). Write the dated delta report:
  `{ReportDir}\{yyyy-MM-dd}-backlog-survey-delta.md` containing a "New Findings" table
  (Id/Section/excerpt), a "Closed Items" table (Id/Section), and a "Trend Counts" section stating
  total current items, new count, closed count, unchanged count, and the `convergence` value in
  prose (referencing REQ-1.5.6 and vNEXT-SEEDS.md's convergence definition by name). If `-Json`
  is set, emit the summary object (`date`, `totalItems`, `newFindings`, `closedItems`,
  `unchangedCount`, `convergence`) to stdout as the sole content (no `Write-Host` noise), matching
  `workspace-health.ps1`'s `-Json` contract. Exit `1` if `-BacklogPath` does not resolve; exit `0`
  otherwise. Keep the file strictly ASCII - no em-dashes, no curly quotes, no non-ASCII characters
  anywhere (this workstation's known PS 5.1 ANSI-decoding hazard).
