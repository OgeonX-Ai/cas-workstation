<objective>
Publish a versioned `FailureState` JSON Schema in cas-contracts (v1.1 line) that gives gsd-orchestrator and autogen MAF workers a shared, implementation-neutral shape for typed failure states, satisfying the "Resilience First" ambiguity resolution in `.planning/REQUIREMENTS.md` (all system errors must map to a defined JSON schema in CAS Contracts).

Purpose: Without this schema, Phase 27's typed failure states in gsd-orchestrator (REQ-1.4.2) and autogen (REQ-1.4.3) would each invent their own shape, defeating the "mapped to CAS Contracts" requirement from ROADMAP.md Phase 27. This plan is wave 1 — plans 27-02 through 27-05 consume the field names defined here (informally, no runtime schema validation dependency, since gsd-orchestrator is C# and autogen is Python — neither directly imports the JSON Schema at runtime, but their emitted structured-log/telemetry field names MUST match this contract).
Output: `schemas/v1.1/failure-state.schema.json`, a `failureClass` shared enum in `common.schema.json`, a valid example, a passing ajv test, and VERSIONING/CHANGELOG entries documenting this as a minor/additive change.
</objective>
