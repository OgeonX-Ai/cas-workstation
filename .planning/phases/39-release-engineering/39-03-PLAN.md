Note on cas-reference-product: the LOCAL clone at
portfolio/cas-reference-product has a tag `v0.1` (`git tag -l`), but `git ls-remote
--tags origin` and `gh api repos/Coding-Autopilot-System/cas-reference-product/tags`
both return empty — this tag was never pushed. Bootstrapping from it would be wrong on
two counts: it's local-only (release-please operates against GitHub via API, so a

not `v0.1.0`). Treat this repo identically to the other zero-tag repos: bootstrap
0.0.0. Do not create, push, or otherwise touch the local `v0.1` tag — out of scope.
