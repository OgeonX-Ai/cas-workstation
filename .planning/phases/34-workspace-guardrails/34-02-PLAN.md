<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI job -> Actions runner filesystem | commit-integrity job requires full git history (`fetch-depth: 0`), increasing checkout time/data on every run |
| register-workspace-health-task.ps1 -> Windows Task Scheduler | Registers a task that runs unattended daily with the invoking user's privileges |
| CI report-only jobs -> merge gate | continue-on-error jobs are visible but do not block; risk of alert fatigue if sweep false-positives frequently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-05 | Denial of Service | workspace-health / commit-integrity CI jobs blocking merges on false positives | mitigate | Both jobs run with `continue-on-error: true` (report-only) per plan constraint; a required-check ratchet is explicitly deferred and documented as a follow-up, not silently dropped |
| T-34-06 | Tampering | register-workspace-health-task.ps1 re-registering / duplicating scheduled tasks on repeat runs | mitigate | `Get-ScheduledTask -ErrorAction SilentlyContinue` idempotency guard exits early if the task name already exists; verified by running the script twice in Task 3's verify step |
| T-34-07 | Elevation of Privilege | Scheduled task runs `pwsh.exe` with whatever privileges the registering user has, unattended | accept | Task only invokes a read-only reporting script (`workspace-health.ps1` performs no writes/mutations to any repo); local single-developer workstation, not a shared/multi-tenant host |
| T-34-08 | Repudiation | commit-integrity heuristic is a simple path-prefix match and can be gamed (e.g. touching an unrelated file matching `tests?/` pattern to satisfy the check without adding real tests) | accept | Documented as a heuristic in CONTEXT.md scope ("Start with a heuristic"); full semantic verification is out of scope for this phase, flagged as a known limitation rather than silently presented as complete |
</threat_model>
