<per_repo_procedure>
For EVERY repo task below, apply this identical procedure (from docs/merge-train-runbook.md + 30-LIVE-STATE.md). `$repo` = `Coding-Autopilot-System/<name>`. Default branch is `main` for all except Promptimprover (`master`).

1. Re-confirm live state per PR: `gh pr view <num> --repo $repo --json number,headRefName,author,reviewDecision,mergeable,statusCheckRollup`.
2. CI GATE: the PR must be GREEN per 30-LIVE-STATE.md. Re-verify with `gh pr checks <num> --repo $repo`. Any REQUIRED check FAILURE/red → STOP that PR, do NOT merge, do NOT fix CI; report as a blocker with the failing-check names. (NEUTRAL+SUCCESS counts as green — dependabot with skipped optional checks.)
3. Merge path:
   - Reviewed/approved OR dependabot-authored → `gh pr merge <num> --repo $repo --squash --delete-branch`.
   - Self-authored + blocked by protection → temp-relax, same sitting, in order:
     a. `gh api -X DELETE "repos/$repo/branches/<default>/protection/enforce_admins"`
     b. `gh pr merge <num> --repo $repo --squash --delete-branch --admin`
     c. `gh api -X POST "repos/$repo/branches/<default>/protection/enforce_admins"` (RE-ENABLE IMMEDIATELY)
   - Substitute `<default>` = `main` (or `master` for Promptimprover).
4. If two PRs on one repo touch the same file, land the first, then `gh pr update-branch <num2> --repo $repo` before merging the second; on conflict, keep BOTH changes.
5. After all of the repo's PRs are handled, if any temp-relax was used verify restoration: `gh api "repos/$repo/branches/<default>/protection/enforce_admins" --jq .enabled` must be `true`.
6. Any permission-classifier denial on a gh/git command → STOP that item, report as blocker, never work around.
</per_repo_procedure>

| Boundary | Description |
|----------|-------------|
| operator → GitHub admin API | per-repo enforce_admins temp-relax is privileged settings mutation |
| PR branch → default branch | unreviewed/red code crossing into main/master |
| gh command → repo target | wrong-repo mis-targeting (esp. two cas-workstation repos) |
| orphan branch → new PR → main | PR-less test-suite branches conflict on ci.yml with dependabot PRs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-30-07 | Repudiation | permission-classifier denial worked around silently | mitigate | per_repo_procedure step 6 mandates STOP-and-report on denial |
| T-30-08 | Tampering | ci.yml conflict resolved by dropping coverage or action bumps | mitigate | Tasks 1 & 2 mandate keeping BOTH changes on conflict; verify coverage steps AND bumped versions present on main |
| T-30-SC | Tampering | n/a — no package installs | accept | Git/GitHub operations only |
